# Design Source Rules

适用范围：`design/` 下全部 `.pen` 设计源与基于画布的协作。

## 一、画布阶段铁律

- `design/frontendMobileScreens.pen` 是消费端页面的最高优先级事实源。
- `design/backendManagement.pen` 是管理后台页面设计的正式事实源。
- 后台管理端一旦进入正式补画阶段：
  - 画布结构、布局、视觉与组件规则统一以下沉到 `docs/rules/backend-management-design-rules.md` 为准。
  - 页面内容语义、模块职责、对象详情与工作台产品规则统一以下沉到 `docs/rules/backend-management-product-rules.md` 为准。
- 所有与设计相关的判断都必须以 `.pen` 的最新实现为准。
- 如果 `.pen` 已发生修改并形成新版本，后续同步、补画、补 schema、补代码时必须站在新版本继续推进。
- 绝对不允许拿旧内容、旧截图、旧 schema、旧摘要去覆盖或回退新的 `.pen`。
- 设计未定稿前，禁止直接进入前端页面开发。
- 只要 `.pen` 仍在调整、补画、讨论或尚未被用户明确确认“可以开始落代码”，就必须先修改画布，不能直接写前端代码。

## 二、画布到 schema 的规则

- 页面设计变化必须先更新 `.pen` 画布，再进入 schema 和代码阶段。
- schema 同步必须通过 Pencil MCP / pencli 直接读取节点结构完成。
- 截图只用于最终校验，不得把视觉识别当成 schema 生成依据。
- 如果 MCP 当前读到的 `.pen` 与旧摘要、旧 schema、旧截图不一致，必须立即以当前 `.pen` 为准校正相关文档，而不是反向把 `.pen` 改回旧状态。
- 本文件只定义画布事实源与设计阶段约束；管理后台画布规则统一以下沉到 `docs/rules/backend-management-design-rules.md` 为准；管理后台页面内容语义统一以下沉到 `docs/rules/backend-management-product-rules.md` 为准；前端代码层面的实现细则统一以下沉到 `docs/rules/frontend-implementation-rules.md` 为准。

## 三、画布修改边界

- 设计不是参考图，必须按最终画布理解。
- 页面结构、模块顺序、弹层形态、状态切换、地图交互、成功反馈都先看 `.pen`。
- 未确认前，不得自行新增、删除、重排模块。
- 新增或补画正式页面时，必须按当前功能模块分区摆放，并尽量贴近实际调用链、入口页或相邻功能页，不能把同一模块页面散落到无关区域。
- 如果必须偏离 `.pen`，只能因为平台能力、性能、合规或安全限制，并且必须先确认。

## 四、画布阶段的设计判断标准

- 进入 `.pen` 设计阶段时，默认必须以“资深产品设计专家”的标准进行判断，不得只做机械画图。
- 每次补画或修改正式页面时，必须同时站在“高级产品经理 + 高级前端开发工程师 + 高级设计师”的复合视角进行判断，不能只从单一美术或单一实现角度出发。
- 每次画图时都必须主动参考业界一些优秀的做法，吸收成熟产品中经过验证的优秀交互、信息组织和视觉处理方式，但不能脱离当前项目的真实调用链、功能语义和既有视觉体系。
- 需要主动审视：
  - 信息层级
  - 操作语义
  - 情绪氛围
  - 视觉秩序
  - 用户决策路径
- 当用户要求继续调整 `.pen` 画布时，默认先做专业设计复盘，再进入具体改画布动作。
- 如果发现结构、主次关系、按钮语义或视觉焦点存在问题，必须先明确指出，不得直接照旧微调。

## 五、补画节奏与确认规则

- 连续补画正式页面时，默认每完成约 3 个页面就暂停一次，先让用户确认是否需要调整，再继续下一批。
- 未经过这一轮确认，不要连续批量补完整个模块，避免后续集中返工。
- 如果用户明确要求连续推进更多页面，可以按用户最新口径执行。

## 六、图片真实度规则

适用范围：

- 商城列表页商品封面
- 商品详情页首屏主图
- 商品详情页正文配图
- 其他消费端页面中承担真实内容表达的图片区域

执行原则：

- 商品图必须服务项目整体气质，不能发展成传统电商促销风。
- 视觉目标统一为：简约、柔和、克制、有呼吸感。
- 推荐背景为白色、暖白、浅米白、浅灰白。
- 商品主体必须明确，画面要有明显留白，不贴边、不满铺。
- 禁止高饱和红橙、彩虹色、强撞色、大面积渐变紫、黑底霓虹感。
- 默认不在图片中放营销文案、价格、角标、参数表。
- `coverUrl` 当前同时承担列表页封面和详情首图，素材必须按单封面模式制作。
- 正文说明图必须通过结构化字段承接，不允许继续依赖模糊图库字段兜底。
- 所有涉及图片展示的正式画布，默认应使用真实图片填充，而不是长时间保留纯色块、空白框或抽象占位。
- 选图时应优先参考当前仓库里已确认页面的图片气质与构图方式，保证同一产品内的图片语言一致。

## 七、小程序导航栏规则

- 适用前提：当前页面属于微信小程序页面，且存在明确的上一级返回页面或返回路径。
- 满足上述前提时，画布顶部应优先参考微信默认导航栏语义来设计，包括左侧返回箭头、中间标题、右侧留白或轻量功能位，不要随意改成无返回语义的自定义头部。
- 首页、一级入口页、Tab 页或其他不存在返回页的页面，不需要额外加入返回箭头。
- 如果页面虽然是二级页，但产品语义上不应返回上一页，必须先确认后才能偏离这条规则。
