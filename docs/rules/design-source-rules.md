# Design Source Rules

适用范围：`design/` 下全部 `.pen` 设计源与基于设计源的协作。

## 核心原则

- `design/frontendMobileScreens.pen` 是消费端页面的最高优先级事实源。
- `design/backendManagement.pen` 当前只有空白画板，不能作为后台开发依据。
- 代码和后端接口都必须服务已确认设计，不能反向用旧实现定义新页面。
- 设计未定稿前，禁止直接进入前端页面开发。

## 设计落地规则

- 设计不是参考图，必须按最终画布理解。
- 页面结构、模块顺序、弹层形态、状态切换、地图交互、成功反馈都先看 `.pen`。
- 未确认前，不得自行新增、删除、重排模块。
- 如果必须偏离 `.pen`，只能因为平台能力、性能、合规或安全限制，并且必须先确认。

## 当前关键设计事实

- 地址链路按“地址管理页 + 地址编辑页”两页版理解。
- 地图选点、搜索建议、粘贴识别都属于地址编辑页内部能力。
- 宠物链当前已包含：
  - `cKCx8` 宠物列表页
  - `H6Xma` 单宠物健康档案页

## 商品图规则

适用范围：

- 商城列表页商品封面
- 商品详情页首屏主图
- 商品详情页正文配图

执行原则：

- 商品图必须服务项目整体气质，不能发展成传统电商促销风。
- 视觉目标统一为：简约、柔和、克制、有呼吸感。
- 推荐背景为白色、暖白、浅米白、浅灰白。
- 商品主体必须明确，画面要有明显留白，不贴边、不满铺。
- 禁止高饱和红橙、彩虹色、强撞色、大面积渐变紫、黑底霓虹感。
- 默认不在图片中放营销文案、价格、角标、参数表。
- `coverUrl` 当前同时承担列表页封面和详情首图，素材必须按单封面模式制作。
- 正文说明图必须通过结构化字段承接，不允许继续依赖模糊图库字段兜底。

## 协作要求

- 设计变化后，必须同步更新根级规则、handoff 与必要契约。
- 不再保留独立旧页面设计口径，例如 `address-picker / D5qTR`。
