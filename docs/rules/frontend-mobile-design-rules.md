# Frontend Mobile Design Rules

适用范围：`design/frontendMobileScreens.pen` 与消费端小程序页面的 `.pen` 设计协作。

## 一、分层边界

- 本文件只定义消费端页面设计规则：
  - 视觉基调
  - 场景色调
  - 图片真实度
  - 页面结构与交互风格
  - 小程序导航栏设计约定
- `.pen` 作为事实源的优先级与协作规则，统一以下沉到 `docs/rules/design-source-rules.md` 为准。
- schema 与代码实现规则，统一以下沉到 `docs/rules/frontend-implementation-rules.md` 为准。

## 二、视觉基调

- 页面统一追求：简约、柔和、有呼吸感、克制但不冷淡。
- 高级感来自秩序、留白、文字层级和轻描边，不来自装饰堆砌。
- 不做花哨运营风、彩虹功能面板、厚重卡片、强渐变堆叠。
- 强调色要少而准，绿色只用于成功状态，不扩散成整页主色。

## 三、场景色调规则

- 品牌内容页：
  - 首页
  - 社区
  - 我的
  - AI 诊断
  - 应保持统一品牌氛围，但具体色相、明度、材质表达以当前 `.pen` 为准，不预设固定紫白方案。
- 交易确认页：
  - 商城
  - 商品详情
  - 购物车
  - 结算
  - 支付相关页面
  - 地址管理
  - 地址编辑
  - 应保持交易链路的克制、中性、稳定感，具体色彩与层级以当前 `.pen` 为准。
- 同一条业务链路中的图片语言、留白方式与情绪氛围必须保持一致，不允许同一产品内同时出现多个互相冲突的视觉体系。

## 四、页面结构与交互风格

- 一个页面只保留一个强焦点，不在首屏同时堆多个同级重点模块。
- 顶部结构做减法，避免为了“设计感”增加无业务价值的装饰头部。
- 卡片要轻，不要厚；优先白底、轻描边、轻阴影。
- 操作区做轻反馈化处理，不做控制面板感。
- 允许克制型微交互，例如轻缩放、轻回弹、轻透明度过渡。
- 不允许大面积粒子、持续悬浮动画、多处同时高存在感动效。

## 五、图片真实度规则

适用范围：

- 商城列表页商品封面
- 商品详情页首屏主图
- 商品详情页正文配图
- 其他消费端页面中承担真实内容表达的图片区域

执行原则：

- 商品图必须服务项目整体气质，不能发展成传统电商促销风。
- 推荐背景为白色、暖白、浅米白、浅灰白。
- 商品主体必须明确，画面要有明显留白，不贴边、不满铺。
- 禁止高饱和红橙、彩虹色、强撞色、大面积渐变紫、黑底霓虹感。
- 默认不在图片中放营销文案、价格、角标、参数表。
- `coverUrl` 当前同时承担列表页封面和详情首图，素材必须按单封面模式制作。
- 正文说明图必须通过结构化字段承接，不允许继续依赖模糊图库字段兜底。
- 所有涉及图片展示的正式画布，默认应使用真实图片填充，而不是长时间保留纯色块、空白框或抽象占位。
- 选图时应优先参考当前仓库里已确认页面的图片气质与构图方式，保证同一产品内的图片语言一致。

## 六、小程序导航栏设计规则

- 适用前提：当前页面属于微信小程序页面，且存在明确的上一级返回页面或返回路径。
- 满足上述前提时，画布顶部应优先参考微信默认导航栏语义来设计，包括左侧返回箭头、中间标题、右侧留白或轻量功能位，不要随意改成无返回语义的自定义头部。
- 首页、一级入口页、Tab 页或其他不存在返回页的页面，不需要额外加入返回箭头。
- 如果页面虽然是二级页，但产品语义上不应返回上一页，必须先确认后才能偏离这条规则。
- 默认不在消费端页面中伪造系统状态栏时间、信号、电量等系统信息。
