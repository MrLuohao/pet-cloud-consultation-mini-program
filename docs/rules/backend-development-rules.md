# Backend Development Rules

适用范围：`apps/backend/pet-cloud-consultation/` 下全部 Java、数据库、接口协作与后台实现契约。

## 一、实现前置原则

- 后端实现必须服务已经确认的页面交互与对象闭环，不允许旧接口反向定义新页面。
- 管理后台相关实现，必须以前置三层事实共同约束：
  - `design/backendManagement.pen`
  - `docs/rules/backend-management-design-rules.md`
  - `docs/rules/backend-management-product-rules.md`
- 如果页面动作、对象关系、状态流转还没有在规则或 `.pen` 中明确，先补规则与画布，再补接口和表结构。

## 二、Java 与编码规范

- 后端编译、测试统一使用 Java 21。
- 注释必须写在代码上方，禁止行尾注释。
- 变更接口后，必须同步 API 文档与根级协作文档。

## 三、数据库规则

- `apps/backend/pet-cloud-consultation/db/pet_cloud_db.sql` 是项目唯一保留的建表 SQL 文件。
- 其他 SQL 迁移脚本不再作为长期保留入口。
- 建表应优先以“最终态建表”维护，不依赖多份历史 ALTER 脚本拼接。

## 四、BaseEntity 规则

- `BaseEntity` 默认只承接通用审计字段：
  - `id`
  - `creator_id`
  - `creator_name`
  - `create_time`
  - `modifier_id`
  - `modifier_name`
  - `modify_time`
- `BaseEntity` 默认不再携带 `is_deleted`。
- 是否需要软删除，由具体实体按业务语义自行声明，不能把软删除当默认字段。

## 五、is_deleted 判断

- 主内容表、可撤销业务表、需要误删恢复或审核下架的表，才考虑保留 `is_deleted`。
- 流水表、事实记录表、审计日志表、关系表、中间表通常不保留 `is_deleted`。
- 双边共享数据不得用单个 `is_deleted` 表达“某一方删除”。
- 私信会话、群会话成员状态、收藏夹展示状态这类按参与人可见性变化的数据，应优先使用“按用户维度状态字段”，例如隐藏时间、归档状态、置顶状态。
- 当前项目内，`community_post`、`community_comment` 保留软删除；`private_conversation` 不再使用全局软删除。

## 六、鉴权规则

- 用户行为、个人数据、交易相关、预约相关接口默认需要登录。
- 内容浏览、公共展示、受控访客体验接口可公开。
- 鉴权策略必须按产品语义判断，不能只图实现方便。

## 七、当前后端协作口径

- 后端接口必须让位于已确认的页面交互。
- 地址链路按地址管理页 + 地址编辑页理解，不再为独立地址选择页维持旧语义。
- `pet_cloud_db.sql` 必须覆盖后端代码当前实际使用的全部表。

## 八、当前接口与字段协作规则

- 首页能力继续围绕：
  - `GET /api/home/summary`
  - `GET /api/pets/current-card`
  - `GET /api/pets`
  - `GET /api/health/reminders/summary`
  - `GET /api/care/today-summary`
  - `GET /api/content/featured`
- 社区能力继续围绕：
  - `GET /api/community/feed`
  - `GET /api/community/posts`
  - `GET /api/community/post/{postId}`
  - `POST /api/community/post`
- 商品详情继续以 `specGroups`、`highlights`、`storySections`、`usageNote` 为结构化契约。
- 购物车、订单确认、订单提交、订单快照必须继续保留 `specLabel/specLabels`。
- 地址保存以编辑页最终字段值为准，不得用历史识别结果反向覆盖。
- `imageUrls` 只作为商品图库，不承担正文展示顺序语义。

## 九、管理后台指标口径

- `内容带成交` 固定按 `24 小时点击归因` 口径计算。
- `综合投诉率` 的分母固定按有效服务交互 / 有效订单计算，不按活跃用户数计算。
- `发货准时率` 固定按承诺发货时间计算，不按订单创建时间计算。
- `售后平均处理时长` 固定按“售后申请提交时间 -> 最终结案时间”计算。

## 十、异常识别框架

- 所有异常识别都必须同时满足：
  - 绝对值或样本量下限
  - 相对 7 日均值的波动判断
- 告警风格统一按 `保守、低噪音` 理解，不允许轻微波动频繁触发红色风险。
- 阈值配置应支持：
  - 系统默认值
  - 业务域覆盖值
- 风险等级统一使用：
  - `关注`
  - `预警`
  - `严重`

## 十一、统一治理与投诉 SLA 契约

- 治理案件严重度统一使用：
  - `普通`
  - `高优`
  - `严重`
- 首响 SLA 固定为：
  - 普通：`4 小时`
  - 高优：`2 小时`
  - 严重：`30 分钟`
- 最终结案 SLA 固定为：
  - 普通：`48 小时`
  - 高优：`24 小时`
  - 严重：`8 小时`
- 统一治理首页的最高优先级告警固定为 `超 SLA 案件`。
- `重复投诉风险` 定义为同类型问题 `7 天内 >= 2 次`。
- `投诉升级异常` 定义为相对 7 日基线 `+30%` 且样本量 `>= 20`。
- `业务域投诉异常` 定义为相对 7 日基线 `+20%` 且样本量 `>= 20`。

## 十二、订单履约异常阈值

- `发货准时率异常` 定义为相对 7 日基线下降 `5 个百分点` 以上，且满足样本量下限。
- `物流异常率异常` 定义为相对 7 日基线提升 `20%` 以上，且满足样本量下限。
- `售后超时异常` 定义为超 SLA 占比较 7 日基线提升 `20%` 以上，且满足样本量下限。
- `升级至治理中心异常` 定义为较 7 日基线提升 `20%` 以上，且满足样本量下限。

## 十三、内容与精选异常阈值

- `精选 CTR 下滑` 定义为较 7 日均值下降 `15%`，且曝光量 `>= 1000`。
- `内容带成交下滑` 定义为较 7 日均值下降 `20%`，且归因订单量 `>= 20`。
- `高潜商品机会` 定义为点击 / 访问提升 `20%` 以上，同时支付转化下降 `15%` 以上。
- `精选位失灵` 定义为 `CTR 下滑` 与 `内容带成交下滑` 同时成立。

## 十四、社区审核异常阈值

- `举报率异常` 定义为较 7 日均值提升 `20%`，且发帖量 `>= 100`。
- `申诉异常` 定义为较 7 日均值提升 `15%`，且审核内容量 `>= 30`。
- `误杀纠偏异常` 定义为较 7 日均值提升 `15%`，且审核内容量 `>= 30`。
- `发帖量偏移` 定义为较 7 日均值偏移 `±20%`；该项只作为生态波动提醒，不直接标红。

## 十五、医生 / 问诊运营异常阈值

- `首响超时异常` 定义为较 7 日均值提升 `20%`，且问诊量 `>= 20`。
- `投诉率异常` 定义为较 7 日均值提升 `20%`，且问诊量 `>= 20`。
- `完结率异常` 定义为较 7 日均值下降 `15%`，且问诊量 `>= 20`。
- `复诊贡献机会` 定义为较 7 日均值提升 `20%`，它属于机会识别，不直接归入风险红线。

## 十六、对象级审计日志契约

- 以下动作必须生成对象级审计日志：
  - 发布
  - 回滚
  - 下线
  - 覆盖式发布
- 日志字段至少包括：
  - `action_type`
  - `object_type`
  - `object_id`
  - `object_name`
  - `before_version`
  - `after_version`
  - `diff_summary`
  - `actor`
  - `action_time`
  - `effective_mode`
  - `effective_time`
  - `status`
  - `impact_scope`
  - `note`
- `diff_summary` 需要同时支持：
  - 业务语义摘要
  - 可展开的字段级 diff
- `回滚`、`强制下线`、`覆盖式发布` 必须填写备注。
- `impact_scope` 必须显式记录影响页面或影响坑位。
- 日志详情必须支持跳转：
  - 当前对象
  - 历史版本

## 十七、导出契约

- 导出 V1 固定以 `Excel` 为主格式。
- 导出语义固定按当前筛选结果生成，不允许默认导出全量数据。
- 导出字段固定按：
  - 核心固定字段
  - 当前页面可见列
- 原始敏感证据默认不导出，只导出摘要字段。
- 用户数据在导出文件中必须脱敏；完整信息只允许在系统详情页内查看。
- 导出文件保留期固定为 `7 天`。
- 系统应提供轻量导出中心，用于查看任务状态与最近导出记录。

## 十八、权限边界

- 内容运营角色可以执行：
  - 预览
  - 排期
  - 发布
- `回滚` 默认只开放给管理员或负责人角色。
- 治理角色可以执行轻量治理动作，但不能默认执行高风险补偿或限制动作。
- 补偿、限制、强制下线、强制覆盖发布等高影响动作必须要求更高权限。
