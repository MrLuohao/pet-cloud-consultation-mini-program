# 单宠物健康档案页设计方案

日期：2026-03-17

## 1. 目标

在当前 `宠物列表页 -> 下游页面` 的基础上，补出一个真正承担“单只宠物持续照护信息”的纵深页面，作为宠物模块下一阶段的核心承接页。

本页面的目标不是做成“宠物主页”或“数据看板”，而是建立一个更专业、稳定、可持续扩展的 `健康档案页`。

## 2. 设计定位

页面定位：

`单宠物健康档案页`

它负责回答两个问题：

- 这只宠物现在状态怎么样
- 接下来需要关注什么、能回看什么

因此，这个页面必须：

- 比宠物列表页更专业
- 比交易页更柔和
- 比首页更克制
- 保持持续照护感，而不是一次性功能页感

## 3. 与现有设计链路的关系

- 当前 `.pen` 中已确认的宠物入口页为：
  - `design/frontendMobileScreens.pen`
  - 画板：`我的宠物列表 My Pets`
  - `frameId = cKCx8`
- 当前 `cKCx8` 已经收敛为“画廊式宠物档案入口页”，只负责：
  - 选择宠物
  - 快速理解当前状态
  - 进入单只宠物的纵深页

结论：

`单宠物健康档案页` 是 `cKCx8` 的自然下游，而不是独立风格的新页面。

## 4. 页面核心方向

本页采用：

`健康摘要优先 + 提醒轻过渡 + 记录纵向沉淀`

不采用以下方向：

- 不做“宠物身份主页优先”
- 不做“大量指标并列的 dashboard”
- 不做“提醒列表压过健康摘要”的结构

原因：

- 用户进入这个页面，第一目标应当是判断宠物当前健康状态
- 页面需要体现专业照护感，而不是情绪化首页感
- 宠物身份信息应作为情感入口存在，但不能抢走健康摘要的首屏主焦点

## 5. 页面结构

页面从上到下按以下顺序固定：

1. `轻身份头部`
2. `强健康摘要卡`
3. `轻提醒概览`
4. `健康记录时间线`
5. `底部动作区`

### 5.1 轻身份头部

作用：

- 告诉用户当前查看的是哪只宠物
- 提供最基本的身份与情感入口

内容建议：

- 宠物头像
- 宠物名
- `品种 · 年龄`
- 一句很短的状态描述或个性描述

约束：

- 不做大 hero
- 不堆叠大量徽章
- 不做多按钮控制面板
- 只允许非常轻的附加状态点缀

### 5.2 强健康摘要卡

这是全页第一视觉重点。

作用：

- 让用户在进入页面第一屏就知道“这只宠物现在怎么样”

内容层级建议：

- 当前状态：
  - `稳定`
  - `需关注`
  - `待完善`
- 最近一次关键记录时间
- 一句简短照护建议

可补充的轻摘要信息：

- 最近记录数
- 待关注项
- 下次提醒

约束：

- 这些摘要信息必须像服务摘要，不像仪表盘 KPI
- 不使用厚重图表
- 不使用后台式模块切分

### 5.3 轻提醒概览

作用：

- 作为“当前状态”到“历史记录”的轻过渡
- 告诉用户当前是否有需要处理的事项

建议表达：

- `当前暂无待处理提醒`
- `2 条提醒待处理`
- 下方补一行很短的类型说明

约束：

- 不做成长列表
- 不做高密度任务清单
- 不抢健康摘要卡的主焦点

### 5.4 健康记录时间线

这是页面主体，负责建立专业沉淀感。

承接内容：

- 疫苗记录
- 驱虫记录
- 就诊记录
- AI 诊断沉淀记录
- 日常观察记录

结构建议：

- 单纵向时间线
- 单列轻卡片
- 每条记录只保留：
  - 时间
  - 类型
  - 简短标题
  - 一句补充说明

约束：

- 不拆复杂 tabs
- 不做数据报表
- 不做满屏标签墙

### 5.5 底部动作区

作用：

- 承接后续操作
- 保证页面继续向下游能力延伸

动作建议：

- 主动作：`新增健康记录`
- 次级动作：
  - `查看成长档案`
  - `编辑宠物资料`

约束：

- 不做三个同权重大按钮
- 主次关系必须明确
- 动作区要轻，不做工具面板感

## 6. 视觉方向

整体视觉采用：

`浅紫白体系 + 轻白卡 + 低饱和 + 克制层级`

风格判断：

- 比首页更安静
- 比交易页更柔和
- 比 AI 诊断页更轻

具体要求：

- 页面底色使用极浅暖白紫
- 卡片以白底、轻描边、极轻阴影为主
- 强调色仅在状态词、轻点缀、主 CTA 中使用
- 不使用大面积强渐变
- 不使用厚重医疗后台风
- 不使用“成长看板”“健康大屏”式视觉表达

## 7. 首屏焦点规则

首屏视觉优先级固定如下：

1. `健康摘要卡`
2. `宠物身份信息`
3. `轻提醒概览`

结论：

用户进入页面后，第一眼必须先看到“当前状态判断”，而不是先被头像、背景图、统计块或提醒列表吸走。

## 8. 与现有后续页面的关系

当前页面只负责“单只宠物的健康档案总入口”，不吞并所有下游能力。

因此：

- `新增健康记录` 继续指向健康记录新增能力
- `查看成长档案` 继续指向成长时间线能力
- `编辑宠物资料` 继续指向宠物资料编辑能力

这意味着本页是“深度入口页”，不是所有信息的终点页。

## 10. 一句话结论

`单宠物健康档案页` 应当被设计成一个“健康状态判断优先、提醒轻过渡、记录可沉淀”的持续照护页面，而不是宠物主页，也不是健康数据看板。

## 11. 首轮视觉精修补记（2026-03-17）

本轮在 `.pen` 首稿基础上，已经额外完成一轮“高级感”方向的视觉精修，后续如补 schema 或落前端时，应按当前精修后的画板理解，而不是按首稿的更厚卡片版本理解。

当前精修后的判断如下：

- 页面标题已收敛，不再使用过强的大标题压住首屏
- 健康摘要卡保留第一焦点，但内部统计信息已减弱为更轻的服务摘要，而不是 KPI 面板
- 提醒概览卡已与时间线卡拉开层级：
  - 提醒卡更暖、更轻
  - 时间线卡更中性、更稳定
- 时间线日期块已减重，避免页面中下部继续出现过强彩色块
- 底部动作区已进一步明确主次：
  - `新增健康记录` 保持主 CTA
  - 两个次级动作改为更轻的辅助入口

结论：

当前画板已经从“结构正确”推进到“层级更稳、气质更克制、整体更高级”的版本。
